# 內容規劃

> 站名(placeholder):**拾光選物**
> 方向:生活風格部落格 + 嚴選選物商店
> 最後更新:與 `網站架構.md` 並列維護

---

## 一、品牌定位

| 項目 | 方向 |
|------|------|
| **客群** | 25–40 歲城市生活者,喜歡質感、慢生活、願意為故事付費 |
| **語氣** | 溫暖、真實、有個人觀點但不做作,像跟朋友說話 |
| **視覺** | 留白多、暖色調、自然光、避開網紅濾鏡感 |
| **Blog 角色** | 內容場:用「閱讀、旅行、選品」建立信任感 |
| **Shop 角色** | 轉化場:把文章裡出現的好物變成可購買的選品 |
| **內容 ↔ 商品連結** | 每篇文末推薦 2–3 件對應商品,商品頁附「出處文章」連結 |

## 四、Schema 規劃(內容上線所需的資料表改動)

> 完整 SQL 寫入 `init-scripts/init.sql`,資料庫第一次啟動時由 MSSQL image 自動執行。

### 4.1 P0 — 內容上線前必修(已寫入 init.sql)

| 表格 | 新增欄位 | 用途 |
|------|---------|------|
| `categories` | `slug NVARCHAR(80) UNIQUE` | 友善 URL `/blog/travel` |
| `posts` | `slug NVARCHAR(200) UNIQUE` | SEO 友善 URL |
| `posts` | `excerpt NVARCHAR(500)` | 列表頁摘要 |
| `posts` | `cover_image_url NVARCHAR(500)` | 封面圖 |
| `posts` | `category_id INT FK -> categories.id` | 文章分類 |
| `posts` | `is_published BIT DEFAULT 1` | 草稿/發佈 |
| `posts` | `published_at DATETIME` | 排程發文 |
| `posts` | `updated_at DATETIME` | 維護追蹤 |
| `products` | `slug NVARCHAR(200) UNIQUE` | SEO 友善 URL |
| `products` | `category_id INT FK -> product_categories.id` | 商品分類 |
| `products` | `image_url NVARCHAR(500)` | 主圖(之後可改多圖) |
| `products` | `is_active BIT DEFAULT 1` | 上/下架 |
| `products` | `created_at DATETIME` | 「新品」排序 |
| `products` | `updated_at DATETIME` | 維護追蹤 |
| `orders` | `order_number NVARCHAR(40) UNIQUE` | 對顧客的單號 |

## 五、CRUD API 規劃(OpenCode 實作)

### 5.1 Blog 服務(`blog:5000`)

| Method | Path | 用途 |
|--------|------|------|
| GET | `/categories` | 列出所有文章分類 |
| POST | `/categories` | 新增分類 |
| GET | `/posts` | 文章列表(支援 `?category=`、`?tag=`、`?is_published=`) |
| GET | `/posts/<slug>` | 取得單篇(用 slug 而非 id) |
| POST | `/posts` | 新增文章(body 含 `title`、`slug`、`content`、`category_id`、`tag_ids[]`) |
| PUT | `/posts/<id>` | 更新文章 |
| DELETE | `/posts/<id>` | 軟刪除(`is_published = 0`) |
| GET | `/tags` | 列出所有 tag |
| GET | `/posts/<id>/products` | 取得該文章推薦的商品 |

### 5.2 Shop 服務(`shop:5001`)

| Method | Path | 用途 |
|--------|------|------|
| GET | `/categories` | 商品分類列表 |
| POST | `/categories` | 新增商品分類 |
| GET | `/products` | 商品列表(支援 `?category=`、`?is_active=`) |
| GET | `/products/<slug>` | 取得單品(用 slug) |
| POST | `/products` | 新增商品 |
| PUT | `/products/<id>` | 更新商品 |
| DELETE | `/products/<id>` | 軟刪除(`is_active = 0`) |
| GET | `/products/<id>/posts` | 取得推薦此商品的文章 |

### 5.3 共通設計

- 所有 POST/PUT/DELETE 不做 auth(MVP 階段;之後再加 JWT 或 session)
- 統一錯誤格式:`{"error": "<message>"}` + 對應 HTTP code
- 統一成功回應:直接回資源 dict,新增動作回 `201`
- 用 `db.session.add()` + `db.session.commit()` 標準 SQLAlchemy 寫入
- 軟刪除(`is_published` / `is_active` 設 0),不做真刪除

## 六、驗收清單

完成後應該可以:

1. `docker compose up -d` 一次到位,`init.sql` 自動建表+塞資料
2. 訪問 `http://localhost/blog/` 看到 Blog 404 → 接下來要加前端
3. `curl http://localhost/blog/posts` 看到 8 篇文章
4. `curl http://localhost/shop/products` 看到 10 件商品
5. `curl -X POST http://localhost/blog/posts -H "Content-Type: application/json" -d '{...}'` 可以新增文章
6. 軟刪除後 GET 列表不再出現
