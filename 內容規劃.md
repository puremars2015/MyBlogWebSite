# 內容規劃

> 站名(placeholder):**拾光選物**
> 方向:生活風格部落格 + 嚴選選物商店 + 巨集經濟儀表板
> 最後更新:與 `網站架構.md` 並列維護

## 一、品牌定位

| 項目 | 方向 |
|------|------|
| **客群** | 25–40 歲城市生活者,喜歡質感、慢生活、願意為故事付費 |
| **語氣** | 溫暖、真實、有個人觀點但不做作,像跟朋友說話 |
| **視覺** | 留白多、暖色調、自然光、避開網紅濾鏡感 |
| **Blog 角色** | 內容場:用「閱讀、旅行、選品」建立信任感 |
| **Shop 角色** | 轉化場:把文章裡出現的好物變成可購買的選品 |
| **Admin 角色** | 後台場:讓編輯團隊管理 Blog / Shop / Dashboard 內容 |
| **Dashboard 角色** | 資料場:提供巨集經濟指標,作為 AI Agent 與編輯選題的輔助 |
| **內容 ↔ 商品連結** | 每篇文末推薦 2–3 件對應商品,商品頁附「出處文章」連結 |

## 五、Dashboard 巨集儀表板

### 5.1 設計目標

- **完全自動化**:每個指標每天由排程自動抓公開資料,沒有手動 mock
- **可讀性**:每張卡片顯示「目前值 + 變化 + 觀測時間 + 冷熱判讀 + 比較基準」
- **可被 AI Agent 讀**:一個 API 一次吐所有指標,免爬蟲
- **狀態透明**:`/scheduler` 頁面把排程健康度攤開,失敗時能看到錯誤訊息

### 5.2 指標清單(SERIES_SEED)

### 5.3 冷熱判讀規則

每個指標有獨立門檻(過冷 / 偏冷 / 中性 / 偏熱 / 過熱),集中存放在 `dashboard/app.py` 的 `TEMPERATURE_RULES`。整體判讀用加權平均 score:
- 整體 score ≤ -1.2 → 整體過冷
- -1.2 < score ≤ -0.35 → 整體偏冷
- -0.35 < score ≤ 0.35 → 整體中性
- 0.35 < score ≤ 1.2 → 整體偏熱
- score > 1.2 → 整體過熱

首頁 hero 會顯示整體判讀,每張卡片顯示該指標的冷熱 tone 與比較基準文字。

### 5.4 AI Agent 端點

```
GET /api/agent/dashboard
```

回傳結構:

- `generated_at`:產生時間(UTC, ISO)
- `note`:使用說明
- `temperature_summary`:整體冷熱摘要(score / label / counts)
- `categories`:依分組整理的指標(`大盤指數`、`市場情緒`、`利率`、`通膨`、`景氣`、`資金流`、`商品`、`台灣市場價格`)
- `items`:所有指標的扁平清單,每個含 `code`、`name`、`description`、`unit`、`source_name`、`value`、`previous_value`、`change_label`、`status_label`、`observed_at`、`has_data`、`temperature`

Agent 端使用方式:定期呼叫 `GET /api/agent/dashboard`,`has_data=false` 的項目表示尚未抓到資料(不該被當作實際指標使用)。

### 5.5 手動 Ingest 端點

- `POST /api/ingest/<code>`(任何序列)
- `POST /api/txf_price` 目前僅 `GET`,`POST` 已停用(改用 TAIFEX 日抓)

當台灣官方總經指標(`taiwan_policy_rate` / `taiwan_cpi_yoy` / `taiwan_gdp_yoy` / `taiwan_unemployment`)的 provider 還沒接好公開來源時,可先以 ingest 端點手動餵入,資料會出現在 dashboard。

## 七、CRUD API 規劃(OpenCode 實作)

### 7.1 Blog 服務(`blog:5000`)

| Method | Path | 用途 |
|--------|------|------|
| GET    | `/categories`              | 列出所有文章分類 |
| POST   | `/categories`              | 新增分類 |
| GET    | `/posts`                   | 文章列表(支援 `?category=`、`?tag=`、`?is_published=`) |
| GET    | `/posts/<slug>`            | 取得單篇(用 slug 而非 id) |
| POST   | `/posts`                   | 新增文章(body 含 `title`、`slug`、`content`、`category_id`、`tag_ids[]`) |
| PUT    | `/posts/<id>`              | 更新文章 |
| DELETE | `/posts/<id>`              | 軟刪除(`is_published = 0`) |
| GET    | `/tags`                    | 列出所有 tag |
| GET    | `/posts/<id>/products`     | 取得該文章推薦的商品 |

### 7.2 Shop 服務(`shop:5001`)

| Method | Path | 用途 |
|--------|------|------|
| GET    | `/categories`              | 商品分類列表 |
| POST   | `/categories`              | 新增商品分類 |
| GET    | `/products`                | 商品列表(支援 `?category=`、`?is_active=`) |
| GET    | `/products/<slug>`         | 取得單品(用 slug) |
| POST   | `/products`                | 新增商品 |
| PUT    | `/products/<id>`           | 更新商品 |
| DELETE | `/products/<id>`           | 軟刪除(`is_active = 0`) |
| GET    | `/products/<id>/posts`     | 取得推薦此商品的文章 |
| GET    | `/orders`                  | 列出訂單(後台用) |
| PUT    | `/orders/<id>`             | 更新訂單狀態 |

### 7.3 Dashboard 服務(`dashboard:5003`)

| Method | Path | 用途 |
|--------|------|------|
| GET    | `/api/summary`             | 儀表板單層摘要(items + 冷熱) |
| GET    | `/api/agent/dashboard`     | **AI Agent 一次讀所有指標** |
| GET    | `/api/txf_price`           | 台指期前一交易日收盤 |
| POST   | `/api/ingest/<code>`       | 手動餵入單一觀測值 |
| GET    | `/scheduler`               | 排程狀態頁(人讀) |

### 7.4 共通設計

- 所有 POST/PUT/DELETE 不做 auth(MVP 階段;之後再加 JWT 或 session)
- 統一錯誤格式:`{"error": "<message>"}` + 對應 HTTP code
- 統一成功回應:直接回資源 dict,新增動作回 `201`
- 用 `db.session.add()` + `db.session.commit()` 標準 SQLAlchemy 寫入
- 軟刪除(`is_published` / `is_active` 設 0),不做真刪除

## 八、驗收清單

完成後應該可以:

1. `docker compose up -d` 一次到位,`init.sql` 自動建表+塞資料
2. 訪問 `http://localhost/blog/` 看到 Blog 頁面
3. `curl http://localhost/blog/posts` 看到 8 篇文章
4. `curl http://localhost/shop/products` 看到 10 件商品
5. `curl -X POST http://localhost/blog/posts -H "Content-Type: application/json" -d '{...}'` 可以新增文章
6. 軟刪除後 GET 列表不再出現
7. `http://localhost:5003/` 看到 25 張巨集指標卡,分頁 tab 可切換
8. `http://localhost:5003/scheduler` 看到所有排程與執行紀錄
9. `curl http://localhost:5003/api/agent/dashboard` 一次拿回所有指標 + 冷熱 summary
10. POST `/api/ingest/<code>` 可手動補台灣總經觀測值
