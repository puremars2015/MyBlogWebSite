# 網站架構

> 最後更新:與 `內容規劃.md` 並列維護

## 概覽

本專案使用 Docker 容器化部署,目前由八個服務組成,涵蓋內容站、選物商城、後台、巨集儀表板、資料排程,以及對外入口的 Cloudflare Tunnel。所有 Flask 服務都透過 gunicorn 提供,並由 Nginx 做路徑基底的反向代理;對外 HTTPS 入口由 `cloudflared` 建立的 Cloudflare Tunnel 提供,免開防火牆 port。

```
   外部使用者 (HTTPS)
        │
        ▼
   ┌──────────────────┐
   │   Cloudflare     │  (Cloudflare edge,免公開主機 IP)
   │   Tunnel (cloudflared)
   └────────┬─────────┘
            │ HTTP/2 over TLS
            ▼
   ┌──────────────────┐
   │     Nginx        │  (內部,只 expose 80)
   │   反向代理         │
   └──────┬───────────┘
          │
   ┌──────┼───────┬───────────┬────────────┐
   │      │       │           │            │
   ▼      ▼       ▼           ▼            ▼
┌────┐ ┌────┐ ┌────┐    ┌──────────┐  ┌────────┐
│Blog│ │Shop│ │Adm.│    │Dashboard │  │  DB    │
│5000│ │5001│ │5002│    │   5003   │  │ MSSQL  │
└────┘ └────┘ └────┘    └────┬─────┘  └────────┘
                              │ ▲
                              │ │
                  ┌───────────┘ │
                  │ dashboard-scheduler │
                  │   (cron-like)      │
                  └────────────────────┘
```

Dashboard 額外有一個 `dashboard-scheduler` 容器,負責定期呼叫各 provider 抓公開資料。

## Container 規劃

| Container              | 用途                                  | 對外 Port | 內部 Port |
|------------------------|---------------------------------------|-----------|-----------|
| `nginx`                | 反向代理,路徑基底分流                  | -         | 80        |
| `cloudflared`          | Cloudflare Tunnel 對外入口            | -         | -         |
| `blog`                 | 部落格 Flask 應用                      | `5000:5000` | 5000    |
| `shop`                 | 選物商城 Flask 應用                    | `5001:5001` | 5001    |
| `admin`                | 後台 Flask 應用                       | `5002:5002` | 5002    |
| `dashboard`            | 巨集儀表板 Flask 應用 + 頁面           | `5003:5003` | 5003    |
| `dashboard-scheduler`  | 排程 worker(同一 image, 跑 `scheduler.py`) | -     | -        |
| `db`                   | SQL Server 2022 Express                 | `1433:1433` | 1433  |

`blog` / `shop` / `admin` / `dashboard` 都同時使用 `expose` 與 `ports` 對外,目的是開發期能直接 `curl http://localhost:<port>/` 略過 Nginx 驗證,正式部署時要記得拿掉 `ports:` 段落。

## Nginx URL 路由(路徑基底)

| URL 路徑       | 目的地       | 說明 |
|----------------|--------------|------|
| `trading.thetainformation.com/*` | `dashboard:5003` | 主站,巨集經濟儀表板 |
| `blog.thetainformation.com/*`    | `blog:5000`      | 部落格所有請求 |
| `shop.thetainformation.com/*`    | `shop:5001`      | 商城所有請求 |
| `admin.thetainformation.com/*`   | `admin:5002`     | 後台所有請求 |
| 預設 fallback (`_` / `*.trycloudflare.com`) | `dashboard:5003` | quick tunnel 模式 + 未設定子網域的 fallback |
| `/health`      | (Nginx 內部) | 健康檢查端點 |

> 對外 HTTPS 入口由 Cloudflare Tunnel 提供,Nginx 內部仍走 HTTP。主站網域是 `trading.thetainformation.com`(dashboard),其他服務用子網域 `*.thetainformation.com`,請在 Cloudflare DNS 把子網域都 CNAME 到 tunnel。

## 環境變數

由 `.env` 帶入,範例請見 `.env-sample`:

| 變數                    | 用途 |
|-------------------------|------|
| `SA_PASSWORD`           | SQL Server sa 密碼 |
| `SESSION_SECRET`        | Flask session 簽章密鑰 |
| `FRED_API_KEY`          | FRED(聯準會經濟資料)API key |
| `DASHBOARD_INGEST_TOKEN`| Dashboard 手動 ingest 端點的 bearer token(留白 = 開發期免 token) |
| `TUNNEL_TOKEN`          | Cloudflare Named Tunnel 的 token(**必填**),從 Cloudflare Zero Trust → Tunnels 取得 |

`FRED_API_KEY` 也會透過 `docker-compose.yml` 注入到 `dashboard-scheduler`,沒有設定時 provider 仍可運作(會回傳錯誤但不中斷其他 job)。

## 對外入口:Cloudflare Tunnel

`cloudflared` 容器是唯一對外的入口,負責建立 Cloudflare Tunnel,連進內部 `nginx:80`。好處:

- 不需要在主機開 80/443 port
- 不需要對外暴露主機 IP
- Cloudflare 自動提供 HTTPS、TLS 憑證、HTTP/2
- 設定 DDoS 防護 / WAF 也都在 Cloudflare 端

預設走 **Named Tunnel**(`cloudflared tunnel run --token $TUNNEL_TOKEN`),需要:
- Cloudflare 帳號 + Zero Trust 建立 named tunnel
- 取得 tunnel token(在 Zero Trust → Tunnels → Install/run 頁面可複製)
- 設定 Public Hostname 把 `trading.thetainformation.com` / `blog.thetainformation.com` / `shop.thetainformation.com` / `admin.thetainformation.com` 都指向 tunnel(Cloudflare 會自動加 DNS CNAME)
- token 貼到 `.env` 的 `TUNNEL_TOKEN=`

如果暫時沒有 token 想測試,可以暫時把 `docker-compose.yml` 中 cloudflared 的 `command` 換成 quick tunnel 模式:
```yaml
command: ["tunnel", "--no-autoupdate", "--url", "http://nginx:80"]
```
啟動後從 `docker compose logs cloudflared` 抓 `*.trycloudflare.com` 臨時網址,僅供本機測試。

Cloudflare 會把真實 client IP 放在 `CF-Connecting-IP` header;`nginx.conf` 已啟用 `set_real_ip_from`(包含所有 Cloudflare 公開 IP 段)+ `real_ip_header CF-Connecting-IP`,讓 Flask 拿到正確來源 IP。

## 排程 (`dashboard-scheduler`)

`dashboard/scheduler.py` 為獨立 worker,與 `dashboard` 容器共用 image,只差在 `command: ["python", "scheduler.py"]`。

- 每 `DASHBOARD_SCHEDULER_POLL_SECONDS` 秒(預設 30)輪詢 `economic_fetch_jobs`
- 透過 `providers/__init__.py` 的 registry 取得對應 provider:
  - `fred` — 利率、CPI、PCE、NFP、GDP、SP500、NASDAQ、VIX
  - `twse_openapi` — 台股加權、外資買賣超、融資餘額
  - `taifex` — 台指期近月前一交易日收盤
  - `cboe` — Put/Call Ratio
  - `cnn_official_fear_greed` — Fear & Greed Index(CNN 官方)
  - `yfinance` — 黃金 / 白銀現貨(目前有 Yahoo rate limit 風險,需監控)
  - `derived` — 2Y-10Y 利差
  - `taiwan_economic` — 台灣央行政策利率 / CPI / GDP / 失業率(目前為手動 ingest fallback)
- 觀測值寫入 `economic_observations`,並在 `economic_fetch_runs` 留下成功/失敗紀錄
- 失敗時 status 為 `error`,message 寫入最近 480 字錯誤訊息,可在 `/scheduler` 頁面看到

## 開發期常用指令

```bash
# 啟動整組服務
docker compose up -d --build

# 重新建立並重啟 dashboard / scheduler / cloudflared
docker compose up -d --build --force-recreate dashboard dashboard-scheduler cloudflared

# 看 cloudflared log,quick tunnel 模式會在這裡印 trycloudflare.com 網址
docker compose logs --tail=200 cloudflared

# 直接打 API / 略過 Nginx(本機開發用)
curl http://localhost:5000/   # blog
curl http://localhost:5001/   # shop
curl http://localhost:5002/   # admin
curl http://localhost:5003/   # dashboard
curl http://localhost:5003/scheduler   # 排程狀態頁
curl http://localhost:5003/api/agent/dashboard | jq .   # AI Agent 端點

# 觸發所有排程立即跑一次
docker compose exec -T dashboard python -c "from app import app, db, EconomicFetchJob, now_gmt8; \
ctx=app.app_context(); ctx.push(); \
EconomicFetchJob.query.update({'next_run_at': now_gmt8()}, synchronize_session=False); \
db.session.commit()"

# 觀察排程 log
docker compose logs --tail=200 dashboard-scheduler
```

## 部署注意事項

- **正式對外走 Cloudflare Tunnel**,nginx 不再對外 expose 80/443。
- 上正式環境前,請在 Cloudflare Zero Trust 建立 Named Tunnel,把 `trading.thetainformation.com` / `blog.thetainformation.com` / `shop.thetainformation.com` / `admin.thetainformation.com` CNAME 到 tunnel,token 貼到 `.env` 的 `TUNNEL_TOKEN`。`docker-compose.yml` 中 cloudflared 已預設走 named tunnel 命令,不用再改。
- `SA_PASSWORD` 與 `SESSION_SECRET` 請改用 secrets 管理。
- `FRED_API_KEY` 有 rate limit,正式部署前請評估流量。
- `yfinance` provider 偶發會被 Yahoo rate limit 影響,排程 log 看到 `YFRateLimitError` 是已知狀況,監控重點為失敗頻率。
