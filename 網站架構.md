# 網站架構

## 概覽

本專案使用 Docker 容器化部署，包含以下服務：
- **Nginx**: 反向代理伺服器
- **Blog**: 部落格 Flask 應用程式
- **Shop**: 商城 Flask 應用程式
- **SQL Server**: 資料庫

```
                    ┌─────────────┐
    使用者 ────────► │    Nginx    │ (port 80)
                    │  反向代理    │
                    └──────┬──────┘
                           │
              ┌────────────┼────────────┐
              │            │            │
              ▼            ▼            ▼
        ┌──────────┐ ┌──────────┐ ┌──────────┐
        │  Blog    │ │  Shop    │ │  DB      │
        │ Flask    │ │ Flask    │ │SQL Server│
        │ port 5000│ │ port 5001│ │ port 1433│
        │(gunicorn)│ │(gunicorn)│ │          │
        └──────────┘ └──────────┘ └──────────┘
```

## Container 規劃

| Container | 用途 | 映射 Port | 環境變數 |
|-----------|------|-----------|----------|
| `nginx` | 反向代理伺服器 | 80, 443 | - |
| `blog` | 部落格 Flask 應用 | 5000:5000 | FLASK_ENV, DB_HOST, DB_PASSWORD |
| `shop` | 商城 Flask 應用 | 5001:5001 | FLASK_ENV, DB_HOST, DB_PASSWORD |
| `db` | SQL Server 資料庫 | 1433:1433 | SA_PASSWORD, ACCEPT_EULA |

## Nginx URL 路由（路徑基底）

Nginx 使用單一 server 區塊，透過路徑基底進行路由分流：

| URL 路徑 | 目的地 | 說明 |
|----------|--------|------|
| `/` | 301 → `/blog/` | 根目錄重新導向至部落格 |
| `/blog/*` | `blog:5000` | 部落格所有請求 |
| `/shop/*` | `shop:5001` | 商城所有請求 |
| `/health` | (Nginx 內部) | 健康檢查端點 |

Nginx 會去除匹配的前綴再轉送給上游服務：
- `/blog/posts` → 轉送為 `/posts` 到 `blog:5000`
- `/shop/products` → 轉送為 `/products` 到 `shop:5001`

## Flask 路由設計

### Blog 服務（port 5000）

| 路由 | 說明 |
|------|------|
| `GET /` | 取得 API 版本資訊 |
| `GET /posts` | 取得所有文章 |
| `GET /posts/<id>` | 取得指定文章 |
| `GET /health` | 健康檢查 |

### Shop 服務（port 5001）

| 路由 | 說明 |
|------|------|
| `GET /` | 取得 API 版本資訊 |
| `GET /products` | 取得所有商品 |
| `GET /products/<id>` | 取得指定商品 |
| `GET /orders` | 取得所有訂單 |
| `GET /health` | 健康檢查 |

## 實作步驟

1. **建立 docker-compose.yml** - 定義 4 個服務，包含 healthcheck 條件等待
2. **建立 Nginx 設定** - 單一 server 區塊，路徑基底路由
3. **建立 Blog Flask 應用** - gunicorn 伺服器，路徑無前綴
4. **建立 Shop Flask 應用** - gunicorn 伺服器，Order 模型含 member_id
5. **建立 SQL Server 初始化腳本** - 掛載至 `/docker-entrypoint-initdb.d/`
6. **設定網路** - 讓 containers 互相溝通
